## Key Decisions & Context

### Why SSH Instead of HTTPS?
1. **No Password Prompts**: SSH key-based auth eliminates need for Personal Access Tokens (PAT) on every push
2. **More Secure**: Private key never transmitted; challenge-response authentication
3. **Platform Independence**: Works on all macOS versions, not dependent on CLI tool availability
4. **Industry Standard**: SSH is the recommended authentication method for Git operations

### Why ED25519 Instead of RSA?
1. **Modern Cryptography**: Smaller keys with equivalent security to 3072-bit RSA
2. **Performance**: Faster key generation and signature verification
3. **GitHub Recommendation**: GitHub explicitly recommends ED25519 for new keys
4. **Future-Proof**: Designed to resist quantum computing attacks better than RSA

## Technical Notes for Future Reference

### SSH Key Management
- Private key location: `~/.ssh/id_ed25519`
- Public key location: `~/.ssh/id_ed25519.pub`
- SSH agent automatically loads key on system startup (if configured)
- To add key manually: `ssh-add ~/.ssh/id_ed25519`

### Git Remote URL Formats
- **SSH** (recommended): `git@github.com:nathanielgiddens/WordRunProject.git`
- **HTTPS** (deprecated for this project): `https://github.com/nathanielgiddens/WordRunProject.git`

### Troubleshooting Commands
```bash
# Verify SSH connection
ssh -T git@github.com

# Check current remote URL
git remote get-url origin

# List SSH keys loaded in agent
ssh-add -l

# View public key (for adding to other services)
cat ~/.ssh/id_ed25519.pub
```
